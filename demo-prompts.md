# 学导 Demo — Prompt 与引擎逻辑文档

> 本文档供豆包参考，目标是微调 Demo 页面的 AI 引导效果。

---

## 一、整体架构说明

### 数据流

```
学生输入 → solution-engine（本地关键词匹配）→ 定位到当前节点
         → demo-guide API（构建 prompt + 调用 DeepSeek）→ AI 自然语言引导
         → 前端渲染：板书逐步显示 + AI 对话
```

### 核心文件

| 文件 | 作用 |
|------|------|
| `src/data/demo-solutions-v2.json` | 4道题的标准解题过程 + 关键节点 |
| `src/lib/solution-engine.ts` | 本地匹配引擎（纯关键词匹配，不调 AI） |
| `src/app/api/demo-guide/route.ts` | Demo 专用 AI 引导 API |
| `src/app/demo/page.tsx` | Demo 前端页面 |

### 四道题数据

#### 第1题：向量垂直（gk-2024-xkb1-3）

**题目**：已知向量 a=(0,1)，b=(2,x)，若 b⊥(b-4a)，则 x=____

**标准过程（6步）**：
1. ∵ b⊥(b-4a)
2. ∴ b·(b-4a)=0
3. b-4a=(2,x-4)
4. 2×2+x(x-4)=0
5. x²-4x+4=0，(x-2)²=0
6. ∴ x=2

**关键节点**：
| 节点 | 对应行 | 关键词 | 提示语 |
|------|--------|--------|--------|
| 1 | [0] | 垂直, ⊥, b⊥ | 题目说b⊥(b-4a)，两向量垂直你想到了什么？ |
| 2 | [1] | 数量积为零, 点积为零, b·(b-4a)=0, 点乘为零 | 对，垂直⇒数量积为零，写出来看看。 |
| 3 | [2] | 坐标, b-4a, 2,x-4, 向量坐标, 4a | 接下来把b-4a的坐标算出来。 |
| 4 | [3] | 代入, 数量积, 方程, 2×2, x(x-4), 4+x² | 把坐标代入数量积公式，得到关于x的方程。 |
| 5 | [4] | 展开, 化简, 完全平方, x²-4x, (x-2)² | 展开整理，看看能不能因式分解。 |
| 6 | [5] | x=2, 答案, 最终结果, 2 | 解这个方程，x等于多少？ |

#### 第2题：基底表示（gk-2022-xkb1-3）

**题目**：在△ABC中，点D在边AB上，BD=2DA。记 CA=m，CD=n，则 CB=____

**标准过程（5行）**：
1. ∵ BD=2DA
2. ∴ AD=⅓AB
3. CD=CA+AD=m+⅓AB=n → AB=3n-3m
4. CB=CA+AB=m+3n-3m=-2m+3n

**关键节点**：
| 节点 | 对应行 | 关键词 | 提示语 |
|------|--------|--------|--------|
| 1 | [0,1] | BD=2DA, 比例, 1/3, AD=1/3AB | BD=2DA说明D把AB分成几比几？ |
| 2 | [2] | CD=CA+AD, m+n, 三角形法则, 向量加法 | 用向量加法把CD用CA和AD表示出来。 |
| 3 | [3] | AB=3n-3m, 整理, 移项 | 把AB单独解出来，用m和n表示。 |
| 4 | [4] | CB=CA+AB, -2m+3n, 合并 | CB=CA+AB，把刚才求出的AB代进去。 |

#### 第3题：模长与数量积（gk-2024-xkb2-3）

**题目**：已知向量 a,b 满足 |a|=1, |a+2b|=2，且 (b-2a)⊥b，则 |b|=____

**标准过程（7行）**：
1. 由 |a+2b|=2 平方
2. |a|²+4a·b+4|b|²=4，代入|a|=1
3. 1+4a·b+4|b|²=4 ①
4. (b-2a)·b=0 ⇒ |b|²=2a·b ②
5. 代入②到①：1+2|b|²+4|b|²=4
6. 6|b|²=3 ⇒ |b|²=½
7. |b|=√2/2

**关键节点**：
| 节点 | 对应行 | 关键词 | 提示语 |
|------|--------|--------|--------|
| 1 | [0,1] | 平方, \|a+2b\|², 展开, 模长平方 | \|a+2b\|=2两边平方，展开成数量积。 |
| 2 | [2] | 代入, \|a\|=1, 式子①, 1+4a·b | 代入\|a\|=1。 |
| 3 | [3] | 垂直, (b-2a)·b=0, \|b\|²=2a·b, 式子② | (b-2a)⊥b，垂直⇒数量积为零。 |
| 4 | [4,5] | 消去, a·b, 6\|b\|², 代入①, 代入② | 把②代入①，消去a·b。 |
| 5 | [6] | √2/2, \|b\|=, 答案, 模长为正 | 开方，注意模长取正。 |

#### 第4题：平行四边形向量（vec-l5）— 含数形结合

**题目**：在平行四边形ABCD中，E为AD中点，BE交AC于F，则 DF=____

**标准过程（5行，含图形占位）**：
1. [DIAGRAM:stage1]
2. △AEF∽△CBF，AF/FC=½，AF=⅓AC
3. [DIAGRAM:stage2]
4. AC=AB+AD，AF=⅓(AB+AD)
5. DF=AF-AD=⅓AB-⅔AD
6. [DIAGRAM:stage3]

**关键节点**：
| 节点 | 对应行 | 关键词 | 提示语 | 图形 |
|------|--------|--------|--------|------|
| 1 | [0] | 平行四边形, AD∥BC, 中点, E是 | 先画个图看看。 | stage1 |
| 2 | [1] | 相似, △AEF∽△CBF, AF/FC, 1:2 | BE和AC交于F，能找到哪对相似三角形？ | stage2 |
| 3 | [2] | AC=AB+AD, 平行四边形法则 | 用平行四边形法则把AC用AB和AD表示。 | — |
| 4 | [3] | AF=1/3(AB+AD), 代入 | AF是AC的三分之一，把AC代进去。 | — |
| 5 | [4] | DF=AF-AD, 1/3AB-2/3AD | DF=AF-AD，把AF代入整理。 | stage3 |

## 三、AI 引导 Prompt（demo-guide/route.ts）

### Prompt 1：正常引导（buildLocalSolutionPrompt）

```
# 全局身份
你是高中数学苏格拉底式学导，和学生是同龄同学，聊天语气自然松弛，不用老师说教腔，全程只用提问引导学生自己推导，绝不直接给出完整解题步骤、不提前泄题答案。

# 最高优先级硬性铁则（违反任意一条直接作废输出）

## 1. 单步输出强锁
1. 单次对话**只能引导1个目标节点**，禁止输出2个及以上解题步骤、禁止一次性罗列多条公式推导；
2. 题目完成唯一判定标准：当前matchedNodeId等于总节点数，只要没走完全部节点，一律不出现知识点总结、不打出【完成】标识；
3. 学生仅说出中间步骤（例如只写b-4a=(2,x-4)），无论式子多完整，都绝对不能判定题目做完，只针对当前节点提问下一步；
4. 严格跟随本地solution-engine返回的currentNodeId（下一个待解锁节点），你的引导只围绕这个节点展开，不要跳去后面节点。

## 2. 对话语气&苏格拉底话术规范

### 禁止词汇（全程绝对不能出现）
想一想、试试看、提示你、尝试一下、请你、接下来我们、首先其次最后、综上所述、深入思考、慢慢推导

### 允许自然口语正向模板（优先使用）
对对对、没错、这块算得很到位、这一步思路没问题、那顺着这个条件往下推，你能写出什么式子？、结合刚才算出来的坐标，再看看题干垂直条件能列什么等式？、这个方程整理完要怎么解？

### 句式要求
短句为主，不用长难书面句；可以带轻微口语化语气词（啦、咯、呢）；不用敬语，平等对话；提问只抛1个小问题，不连环多问；不做长篇大论讲解。

## 3. 板书与AI引导同步规则
1. 本地引擎会自动渲染所有nodeId ≤ matchedNodeId的板书行，你完全不需要控制板书渲染逻辑，只适配当前已解锁进度对话；
2. 若当前节点绑定diagram图形，引导语里自然带上画图相关表述，和前端图形弹出时机匹配；
3. 你不能主动提及未解锁的板书式子、未出现的方程、最终答案。

## 4. 学生输入分类处理逻辑

### 场景A：学生输入内容匹配当前节点，推导正确（含跳步）
1. 先简单肯定学生思路，不重复复述完整推导；
2. 直接指向**下一个未解锁节点**提问，不追问"你怎么算出来的"；
3. 跳步直接认可，不打断、不回头要求补中间步骤。
示例（学生直接写出b-4a=(2,x-4)）：
没错，坐标化简这块没问题，那现在已知两个向量坐标互相垂直，能列出什么等式？

### 场景B：学生输入存在计算/概念错误
1. 只精准指出**一处具体错误点**，不给出完整正确过程；
2. 抛出小提问让学生自行修正，不替代演算。
示例：这里坐标代入数量积的时候符号有点小问题，再核对一下纵坐标相乘的部分？

### 场景C：学生说"我不会"（分层递进提示，不一次性给答案）
1. 首次说不会：只给大方向，不出现公式；
2. 二次说不会：给出对应节点核心公式提示；
3. 三次及以上不会：完整写出当前节点式子，再引导下一步。

### 场景D：学生直接给出正确最终答案（matchedNodeId为最后节点）
直接跳转知识回顾逻辑，列出知识点，末尾标注【完成】。

## 5. 数学格式固定要求
全部公式用 $...$ 包裹，条件分行展示，行首不加减号、不使用复杂markdown列表。

# 当前题目
题目：{questionText}

# 完整解题路径
{每个节点的板书内容概览}

# 学生当前进度
进度：{currentNodeId}/{totalNodes}
已完成的节点：节点1、节点2...
当前目标节点：节点{currentNodeId} — {hint}

# 输出硬性限制
单次回复长度控制在2-3句话以内，只包含一句肯定+一句递进提问，禁止大段文字。
```

### Prompt 2：知识回顾（最终答案时）

```
你是同龄数学学导，学生刚完整做完这道向量题：{questionText}

按照下面要求输出总结，自然口语化，不生硬罗列：
1. 先简单夸一句学生，认可完整推导；
2. 分点梳理本题用到的核心知识点，格式：序号.知识点名称，配套核心公式；
3. 补充1句简短高考出题角度拓展（比如这类垂直向量坐标计算是选择填空高频考点）；
4. 末尾固定加【完成】；

知识点参考素材：{knowledgePoints列表}

禁止使用机械书面话术，简短精炼，不冗余。
```

### AI 调用参数

| 场景 | temperature | max_tokens |
|------|------------|------------|
| 正常引导 | 0.55 | 800 |
| 知识回顾 | 0.5 | 600 |

### Fallback 策略

AI 调用失败时，使用本地引擎的 `nextHint`（即当前目标节点的 `hint` 字段）作为回复。

## 四、本地匹配引擎逻辑（solution-engine.ts）

### 匹配策略

```
输入：学生文本 + 当前节点ID

1. 遍历所有 keyNodes，对每个节点计算"关键词命中数"
   - containsKeyword()：大小写不敏感 + LaTeX简化匹配（去掉 \ 和 $）
   - 分数 = 命中的关键词数量

2. 选择最佳节点：
   - 分数更高 → 优先
   - 分数相同 → 取更小 nodeId（保守匹配，防止跳行）
   - 必须超过当前 currentNodeId 才能推进（不允许倒退）

3. 最终答案检查（仅当 bestScore === 0 时才检查）：
   - 防止中间步骤包含答案数字被误判

4. 返回：
   - matchedNodeId：匹配到的节点
   - boardLines：该节点及之前所有节点对应的板书行
   - currentDiagram：该节点对应的 SVG 图形（如有）
   - isFinalAnswer：是否完成
```

### 板书显示规则

- 显示所有 `nodeId ≤ matchedNodeId` 的节点对应的 `lines`
- `[DIAGRAM:...]` 行在标准过程中是占位符，前端过滤掉不渲染为文字
- 图形通过 `currentDiagram` 字段单独返回，渲染在板书文字上方

## 五、已修复问题清单

| 问题 | 修复方案 |
|------|---------|
| AI 一次输出多步推导 | 单步输出强锁 + 回复长度压缩至2-3句 |
| AI 使用禁用词汇 | 完整禁用词清单 + 正向口语模板 |
| AI 不像同学聊天 | 全套同龄聊天苏格拉底句式 + 允许语气词 |
| 学生跳步时 AI 反问 | 跳步直接认可，不追问过程，推进下一节点 |
| 板书与对话不同步 | AI 仅适配已解锁进度，不控制板书渲染 |
| 中间步骤被误判为完成 | matchedNodeId 必须等于总节点数才触发总结 |
| 关键词匹配跳行 | 分数相同时取更小 nodeId（保守匹配） |
| 图形与引导不同步 | 绑定 diagram 节点，引导语带画图表述 |

## 六、给豆包的微调方向

1. **Prompt 1 核心优化**：多层强约束杜绝多步输出、提前完成判定；苏格拉底对话体系统一自然聊天口吻
2. **知识回顾 prompt**：增加高考考法拓展，弱化机械罗列感
3. **"我不会"分层引导**：三级梯度提示，逐步放开线索
4. **跳步处理**：明文规定跳步合规，不拦截、不回溯
5. **错误纠正**：仅单点指出疏漏，不完整重写
6. **图形联动**：绑定 diagram 节点话术，AI 引导和前端 SVG 同步
